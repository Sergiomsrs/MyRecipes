# 11. Roadmap

## 11.1 Objetivo

El desarrollo de MyRecipes seguirá un enfoque incremental basado en versiones.

Cada versión incorporará un conjunto reducido de funcionalidades completamente terminadas antes de comenzar la siguiente.

Este enfoque permitirá mantener una aplicación estable, facilitar el aprendizaje durante el desarrollo y evitar el crecimiento descontrolado del alcance del proyecto.

---

# 11.2 Filosofía

La prioridad del proyecto no será incorporar el mayor número posible de funcionalidades.

El objetivo será construir una aplicación sólida, mantenible y bien diseñada.

Cada nueva funcionalidad deberá aportar un valor real al usuario y encajar con la filosofía principal de la aplicación:

> Ayudar a las personas a conservar y mejorar sus propias recetas mediante un sistema de versiones.

---

# 11.3 Versión 1.0 (MVP)

La primera versión tendrá como objetivo disponer de una aplicación completamente funcional.

Incluirá las siguientes características.

## Usuarios

- Registro.
- Inicio de sesión.
- Cierre de sesión.
- Gestión básica del perfil.

---

## Recetas

- Crear receta.
- Editar información general.
- Eliminar receta.
- Listado de recetas.

---

## Versiones

- Creación automática de la versión inicial.
- Crear nuevas versiones.
- Historial de versiones.
- Consulta de versiones anteriores.

---

## Contenido

- Ingredientes.
- Pasos.
- Notas.
- Valoración.
- Fotografías.

---

## Infraestructura

- API REST.
- Autenticación JWT.
- PostgreSQL.
- Supabase Storage.
- Despliegue en producción.

# 11.9 Mejoras técnicas

Además de las funcionalidades visibles para el usuario, el proyecto evolucionará técnicamente mediante:

- Mayor cobertura de tests.
- Automatización del despliegue.
- Docker.
- GitHub Actions.
- Monitorización.
- Optimización del rendimiento.
- Refactorización continua.
- Mejoras de accesibilidad.

---

# 11.10 Estado actual

Actualmente el proyecto se encuentra en fase de diseño y planificación.

Durante esta fase se está definiendo:

- La visión del producto.
- La arquitectura.
- El modelo de datos.
- La API REST.
- La estrategia de seguridad.
- La estrategia de testing.
- La infraestructura de despliegue.

Una vez finalizada esta documentación comenzará el desarrollo de la primera versión funcional.

---

# 11.11 Criterio de finalización

Una versión únicamente se considerará terminada cuando cumpla los siguientes requisitos:

- Funcionalidad completamente implementada.
- Código revisado.
- Pruebas ejecutadas correctamente.
- Documentación actualizada.
- Despliegue operativo.
